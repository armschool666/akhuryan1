# Փանիկի միջնակարգ դպրոցի կայք - աշխատանքային նշումներ

Այս ֆայլում ամփոփված է մեր քննարկումը և հաստատված պահանջները, որպեսզի նոր կայքի մշակումը շարունակվի առանց կարևոր տվյալների կորստի։

## Գլխավոր նպատակը

Ստեղծել դպրոցական կայք, որը կծառայի որպես գործնական հարթակ ծնողների, աշակերտների և ուսուցիչների համար։

Կայքը պետք է լինի հայերեն, հասկանալի կառուցվածքով, հեշտ թարմացվող և հարմար ինչպես համակարգչով, այնպես էլ սմարթֆոնով օգտագործելու համար։

## Գործող կայքը

Գործող հասցե՝ https://panik.schoolsite.am/

Ստուգմամբ պարզվեց՝

- Կայքը WordPress է։
- Օգտագործվող theme-ը՝ `twentytwelve`։
- Լեզուն՝ `hy-AM`, UTF-8։
- SEO title-ում երևում էր default tagline՝ «Եւս մեկ կայք Armenian Schools Sites ցանցում»։
- REST API pages endpoint-ը փակ էր 403-ով։
- Sitemap-ը հասանելի էր՝ https://panik.schoolsite.am/sitemap.xml։
- `panik.schoolsite.am`-ը առանձին domain չէ, այլ `schoolsite.am` ենթադոմեյն։
- DNS՝ `panik.schoolsite.am -> 91.205.135.226`։
- `schoolsite.am` nameservers՝ `ns1.ktak.am`, `ns1.schools.am`։

## Հաշվետվություններ

- 2024-2025 ուստարի
- 2018-2025 հաշվետվություններ
- Արխիվային եռամսյակներ
- Առաջին եռամսյակ
- 2-րդ եռամսյակ
- 3-րդ եռամսյակ
- 4-րդ եռամսյակ

- Հաշվետվություններ -> Դպրոցի մասին / Հաշվետվություններ
- ՀԱՇՎԵՏՎՈւԹՅՈւՆՆԵՐ 2018-2025 -> Հաշվետվություններ էջի առանձին բաժին
- Առաջին եռամսյակ -> Հաշվետվություններ էջի բաժին

Որոշվեց ունենալ մեկ ընդհանուր parent բաժին՝ «Խորհուրդներ»։

## Vercel / նոր կայքի նկատառում

Քանի որ `panik.schoolsite.am` ենթադոմեյն է, Vercel տեղափոխելու համար պետք է `schoolsite.am` DNS-ի ադմինների մասնակցությունը։

Հնարավոր տարբերակներ՝

- Schoolsite.am ադմինները փոխում են DNS record-ը դեպի Vercel։
- Գնվում է առանձին domain։
- Նոր կայքը սկզբում տեղադրվում է Vercel-ի preview կամ `*.vercel.app` հասցեով։

Կասկածելի/վատ վերնագրով WordPress էջերը դեռ պետք է ձեռքով ստուգել։ Դրանք չպետք է ջնջել մինչև պարզվի ինչ են։

### Վերջնական տեսանելի menu

Տեսանելի menu-ից հանվել են `Արխիվ` և `Ադմին` հղումները։

Ադմին պանելը մնում է հասանելի միայն ուղիղ URL-ով՝ `/admin`։

Տեսանելի menu-ն հիմա՝

- Գլխավոր
- Դպրոցի մասին
  - Հայտարարություններ
  - Հաշվետվություններ
- Խորհուրդներ
  - Միացյալ կառավարման խորհուրդ
  - Մանկավարժական խորհուրդ
  - Ծնողական խորհուրդ
  - Աշակերտական խորհուրդ
  - Մ/Մ արձանագրություններ
- Ուսումնական գործընթաց
  - Քննություններ
  - Նախագծային աշխատանքներ
- Դպրոցի անցուդարձ
  - Նորություններ
  - Միջոցառումներ
- Կապ
  - Հետադարձ կապ
  - Հասցե և քարտեզ

Մնացած բաժինները պահված են համակարգում, բայց չեն երևում menu-ում, մինչև բովանդակություն ունենան։

### Էջերի կառուցվածք

Dropdown-ի բոլոր ենթակետերը առանձին էջեր են, օրինակ՝

- `/section/about/announcements`
- `/section/about/reports`
- `/section/councils/parent`
- `/section/councils/student`
- `/section/learning/exams`
- `/section/learning/projects`
- `/section/events/news`
- `/section/events/events`
- `/section/contact/feedback`
- `/section/contact/map`

Յուրաքանչյուր child էջ ունի իր առանձին body-ն։

### Ադմին պանել

Ադմին պանելն արդեն localStorage prototype չէ։

Օգտագործվում են API route-եր՝

- `GET /api/materials`
- `POST /api/materials`
- `PUT /api/materials`
- `DELETE /api/materials?id=...`
- `POST /api/upload`

Նյութերը պահվում են՝

`data/materials.json`

Upload ֆայլերը պահվում են՝

`public/uploads`

Ադմինում կա՝

- Բաժին ընտրել
- Էջ ընտրել
- Վերնագիր
- Բովանդակություն
- Upload ֆայլեր
- Արդեն առցանց ֆայլերի հղումներ
- Խմբագրել
- Հեռացնել confirmation-ով
- Ցույց տալ միայն ընտրված էջի նյութերը
- Դիտել ընտրված էջը preview հղում

JSON export-ը ադմինից հանված է։

### Ֆայլերի դասավորում

Ֆայլերը public էջերում խմբավորվում են ըստ տարեթվի։

Եթե ֆայլի անունում կամ URL-ում կա `2022`, `2023`, `2024`, `2025`, `2026` և այլն, այն ընկնում է այդ տարվա տակ։

Եթե տարեթիվ չի գտնվում, ցուցադրվում է՝

`Տարեթիվ չնշված`

### Դպրոցի անցուդարձ

Դպրոցի անցուդարձի համար հաստատված են հիմնական էջերը՝

- Նորություններ
- Միջոցառումներ

Առաջարկված է հետագայում այդ բաժնի համար ունենալ ավելի հարմար layout՝ news/media card-երով, պատկերասրահի և տեսանյութերի հատուկ տեսքով։

### WordPress-ում աշխատելու կանոն

Եթե WordPress-ում ստեղծվի ժամանակավոր user, օրինակ՝ `codex-temp`, աշխատանքային կանոնները՝

- Ոչինչ չջնջել
- Գործող էջերի բովանդակությունը չհեռացնել
- Ֆայլեր չջնջել
- Menu-ն live-ում չփոխել առանց հաստատման
- Theme/plugin չփոխել առանց հաստատման
- Փորձերը անել միայն թեստային էջերում կամ սահմանափակ CSS-ով

WordPress-ի user/password-ը չպետք է ուղարկել բաց chat-ում։ Եթե պետք է փոխանցել, օգտագործել secure one-time secret link։

### WordPress CSS

WordPress-ի համար պատրաստվել է CSS տարբերակ `Twenty Twelve` theme-ի վրա օգտագործելու համար։

Այն պետք է տեղադրվի՝

`Appearance > Customize > Additional CSS`

Այդ CSS-ը փոխում է ընդհանուր տեսքը՝ header, menu, content, files, tables, images, widgets, footer և mobile spacing։

### Կարևոր production նշում

Նոր Next.js կայքի ներկա backend-ը local/server ֆայլային պահոց է։

Vercel deploy-ի դեպքում `data/materials.json` և `public/uploads` մշտական պահոց չեն համարվում։

Production-ի համար պետք է անցնել իրական storage-ի՝ օրինակ՝

- Vercel Blob կամ Supabase Storage ֆայլերի համար
- Supabase, Neon կամ Vercel Postgres նյութերի համար

API route-երը արդեն կան, այնպես որ production storage-ին անցնելիս պետք է հիմնականում փոխել պահոցի implementation-ը։
